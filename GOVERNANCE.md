# NexusAI ERP Governance

## Project Structure

### Roles

#### Maintainers
- Review and merge pull requests
- Release new versions
- Manage project direction
- Enforce code of conduct

#### Contributors
- Submit bug reports and feature requests
- Create pull requests
- Participate in discussions
- Help with documentation

#### Users
- Report bugs and issues
- Suggest features
- Provide feedback

### Decision Making

1. **Technical Decisions**: Made by maintainers with community input
2. **Feature Additions**: Discussed in issues before implementation
3. **Breaking Changes**: Require maintainer consensus
4. **Roadmap**: Published quarterly with community input

### Becoming a Maintainer

Contributors who demonstrate:
- Consistent quality contributions
- Understanding of project architecture
- Constructive community participation
- Commitment to project goals

May be invited to become maintainers.

## Communication

- **GitHub Issues**: Bug reports, feature requests
- **GitHub Discussions**: General questions, ideas
- **Pull Requests**: Code contributions

## Versioning

We follow Semantic Versioning (SemVer):
- MAJOR: Breaking changes
- MINOR: New features (backward compatible)
- PATCH: Bug fixes
