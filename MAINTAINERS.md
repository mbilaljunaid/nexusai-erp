## Maintainer Responsibilities

1. **Code Review**: Review and approve pull requests
2. **Issue Triage**: Categorize and prioritize issues
3. **Releases**: Manage version releases
4. **Documentation**: Keep docs up to date
5. **Community**: Foster inclusive community
6. **Security**: Address security concerns promptly

## How to Become a Maintainer

We welcome new maintainers! To be considered:

1. Make consistent, quality contributions
2. Demonstrate understanding of the codebase
3. Help others in issues and discussions
4. Follow the code of conduct
5. Express interest to current maintainers
